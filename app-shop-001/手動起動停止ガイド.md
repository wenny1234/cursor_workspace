# app-shop-001 手動起動・停止ガイド

このドキュメントでは、`mobile-app`・`mobile-app-backend`・`admin-shop-manage`・`wms-shop-manage` の4つのアプリを、手動で起動・停止する手順を説明します。

---

## 1. システム構成

| アプリ | ディレクトリ | ポート | URL | 説明 |
|--------|-------------|--------|-----|------|
| mobile-app-backend | `mobile-app-backend/` | **8080** | http://localhost:8080/api | モバイルアプリ用 REST API（WMS API 含む） |
| admin-shop-manage | `admin-shop-manage/` | **8081** | http://localhost:8081 | 管理画面（商品・ユーザー管理） |
| wms-shop-manage | `wms-shop-manage/` | **3000** | http://localhost:3000 | WMS 出荷管理（React 18） |
| mobile-app | `mobile-app/` | **4200** | http://localhost:4200 | Angular フロントエンド |

**起動順序（推奨）**

1. Oracle データベース（バックエンド2つが Oracle を使う場合）
2. `mobile-app-backend`
3. `admin-shop-manage`（管理画面を使う場合）
4. `wms-shop-manage`（出荷処理を使う場合）
5. `mobile-app`

## 2. 事前準備

### 必要なソフトウェア

| ソフトウェア | 用途 |
|-------------|------|
| Java 11 以上 | Spring Boot アプリ |
| Maven | バックエンド・管理画面のビルド |
| Node.js 18 以上 | mobile-app・wms-shop-manage |
| npm | フロントエンドの依存関係 |
| Oracle XE（Docker 等） | 本番相当の DB 利用時 |

### 初回のみ：フロントエンドの依存関係インストール

```powershell
cd d:\cursor_workspace\app-shop-001\mobile-app
npm install

cd d:\cursor_workspace\app-shop-001\wms-shop-manage
npm install
```

### データベース（Oracle 利用時）

両バックエンドはデフォルトで Oracle プロファイルを使用します。Oracle が起動していること、スキーマが作成済みであることを確認してください。

```text
database/oracle/02_schema.sql      … テーブル作成
database/oracle/04_orders_schema.sql … 注文テーブル
database/oracle/06_admin_extensions.sql … 管理画面用拡張（任意）
database/oracle/07_wms_shipping.sql … WMS 出荷用（送り場所カラム追加）
```

**WMS 出荷機能を使う場合**は、注文テーブル作成後に `07_wms_shipping.sql` を実行してください。

```powershell
# SQL ファイルをコンテナにコピーしてから実行
docker cp d:\cursor_workspace\app-shop-001\database\oracle\07_wms_shipping.sql oracle-db:/tmp/07_wms_shipping.sql
docker exec -i oracle-db sqlplus -s shop/ShopPassword123@XEPDB1 @/tmp/07_wms_shipping.sql
```

**WSL 上で Docker を使っている場合**（Windows の `docker` コマンドが使えないとき）:

```powershell
wsl -e sh /mnt/d/cursor_workspace/app-shop-001/database/scripts/run-oracle-wms-sql.sh
```

> 既に `shipping_address` カラムがある場合は `ORA-01430` エラーになります（スキップして問題ありません）。

Oracle 接続情報（デフォルト）:

- URL: `jdbc:oracle:thin:@//localhost:1521/XEPDB1`
- ユーザー: `shop`
- パスワード: `ShopPassword123`

## 3. 起動手順

**それぞれ別のターミナル（PowerShell または cmd）で実行してください。**

### 3.1 mobile-app-backend（API）

```powershell
cd d:\cursor_workspace\app-shop-001\mobile-app-backend
$env:SPRING_PROFILES_ACTIVE = "oracle"
mvn spring-boot:run
```

起動確認: ログに `Tomcat started on port(s): 8080` と表示されること。

- API ベース URL: http://localhost:8080/api

> CSV モードで起動する場合は `SPRING_PROFILES_ACTIVE` を `csv` に変更（デフォルト設定）。

---

### 3.2 admin-shop-manage（管理画面）

```powershell
cd d:\cursor_workspace\app-shop-001\admin-shop-manage
$env:SPRING_PROFILES_ACTIVE = "oracle"
mvn spring-boot:run
```

起動確認: ログに `Tomcat started on port(s): 8081` と表示されること。

- ログイン画面: http://localhost:8081/login

---

### 3.3 wms-shop-manage（WMS 出荷管理）

```powershell
cd d:\cursor_workspace\app-shop-001\wms-shop-manage
npm run dev
```

起動確認: ログに `Local: http://localhost:3000/` と表示されること。

- WMS 画面: http://localhost:3000
- `mobile-app-backend` の WMS API（`/api/wms/**`）に接続します。先にバックエンドを起動してください。
- **ADMIN** または **STAFF** でログイン（例: `staff` / `staff123`）

**業務フロー**

1. 「出荷待ち（支払済）」タブで `PAID` 注文を確認
2. 「出荷する」→ 送り場所を入力 → 「出荷済みにする」
3. 注文ステータスが `SHIPPING`（mobile-app では「配送中」）に更新される

---

### 3.4 mobile-app（フロントエンド）

```powershell
cd d:\cursor_workspace\app-shop-001\mobile-app
npm start
```

起動確認: ログに `Angular Live Development Server is listening on localhost:4200` と表示されること。

- アプリ URL: http://localhost:4200

> `mobile-app` は `http://localhost:8080` の API に接続します。先に `mobile-app-backend` を起動してください。

## 4. 停止手順

### 方法 A：ターミナルで停止（推奨）

各アプリを起動したターミナルで **`Ctrl + C`** を押します。

### 方法 B：ポート番号でプロセスを停止（Windows）

ターミナルを閉じてしまった場合など、ポートでプロセスを特定して停止します。

**1. ポートを使用している PID を確認**

```powershell
netstat -ano | findstr ":8080 :8081 :3000 :4200" | findstr LISTENING
```

出力の最右列が PID です。

**2. プロセスを停止**

```powershell
# 例: PID が 12345 の場合
taskkill /PID 12345 /F
```

**ポートごとに一括停止（PowerShell）**

```powershell
foreach ($port in 8080, 8081, 3000, 4200) {
    $conn = Get-NetTCPConnection -LocalPort $port -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1
    if ($conn) {
        Stop-Process -Id $conn.OwningProcess -Force
        Write-Host "ポート $port を停止しました (PID $($conn.OwningProcess))"
    } else {
        Write-Host "ポート $port は使用されていません"
    }
}
```

**停止確認**

```powershell
netstat -ano | findstr ":8080 :8081 :3000 :4200" | findstr LISTENING
```

何も表示されなければ、4つのポートはすべて解放されています。

## 6. よくある問題

### ポートが既に使用中

```
Port 8080 was already in use
```

→ 上記「4. 停止手順」の方法 B で該当ポートのプロセスを停止してから、再度起動してください。

### 管理画面でログインできない

- ユーザー名・パスワードを確認（`admin` / `admin123`）
- Oracle が起動しているか確認
- ログに `ORA-00904` など DB エラーが出ていないか確認

### mobile-app で商品が表示されない

- `mobile-app-backend` が起動しているか確認（http://localhost:8080/api）
- ブラウザでログイン済みか確認（商品 API は認証が必要）

### WMS で出荷処理ができない

- `mobile-app-backend` が起動しているか確認
- `staff` / `staff123` など STAFF 以上でログインしているか確認
- Oracle 利用時は `07_wms_shipping.sql` を実行済みか確認（`ORA-00904: "SHIPPING_ADDRESS"` が出る場合は未実行）

### Oracle に接続できない

- Docker の Oracle コンテナが起動しているか確認
- 接続情報（ホスト `localhost:1521`、ユーザー `shop`）を確認
